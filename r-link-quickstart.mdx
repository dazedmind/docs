## Get started in three steps

Get your documentation site running locally and make your first customization.

### Step 1: Clone the GitHub repository

<AccordionGroup>
  <Accordion title="Clone the website locally" icon="copy">
    1. GitHub repository: [https://github.com/dazedmind/rlink](https://github.com/dazedmind/rlink)
    2. Open cmd and run: `git clone https://github.com/dazedmind/rlink`
    3. Type `cd rlink`
  </Accordion>
</AccordionGroup>

<Steps>
  <Step title="Install npm packages">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Run local development server">
    Navigate to your docs directory where your `docs.json` file is located, and run the following command:

    ```bash
    npm run dev
    ```

    A local preview of your documentation will be available at `http://localhost:3000`.
  </Step>
</Steps>

### Step 2: Setup Environment Variable

<AccordionGroup>
  <Accordion title="Ensure these Environment Variables is set" icon="code">
    ```text
      DATABASE_URL= Neon PostgreSQL connection string (required for `lib/db.ts` and Drizzle)
      NEXT_PUBLIC_APP_URL= Public site URL (e.g. `http://localhost:3000` in dev); used by the auth client
      BETTER_AUTH_URL= Optional; fallback for app origin in some email/config paths
      BETTER_AUTH_SECRET= Secret key from Better Auth
      SITE_URL= Used by health checks and cache-related tooling
      REVALIDATION_SECRET= Bearer secret for developer cache-clear API
      ALLOWED_ORIGIN= Production browser origin for API CORS (defaults in code exist for dev / a dev deploy host)
      RESEND_API_KEY= Sending transactional and campaign email
      RESEND_FROM= Default “from” address (falls back to a Resend onboarding sender if unset)
      CRON_SECRET= Bearer secret for `/api/cron/activity-logs-retention` (Vercel Cron)
    ```
  </Accordion>
</AccordionGroup>

## Backend Setup

The database schema can be found at the `/db` folder

- `schema.ts` - contains the tables and enums for the system
- `auth-schema.ts` - contains specific tables and enums for [Better Auth](https://better-auth.com/docs/introduction)

### Run Drizzle Studio

Go to the project base directory and run:

```text
npx drizzle-kit studio
```

Then Ctrl \+ click  [https://local.drizzle.studio](https://local.drizzle.studio) from the terminal
